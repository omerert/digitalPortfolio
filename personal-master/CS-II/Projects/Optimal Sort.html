Explanation:
This is the graph for the time it takes to sort different word counts with bubble sort. Bubble sort essentially goes from left to right sorting each number. It is generally seen as a slow sorting method due to its inefficient and time-consuming sorting method. While the slope for reversed and random are faily similar, ordered was a little different with a faster time. While bubble sort is efficient when sorting 10e2 to 10e3, it is not practical for bigger counts like 10e5. Therefore, many do not reccomend bubble sort due to its impractical sorting time for large files.

Explanation:
For insertion sort, there were varying slopes: reversed being the slowest and ordered being the fastest. This is due to how the insertion sort algorithm works: sorts from left to right. So for word files like reversed, it is slow since everyting on the right has to go to the left, taking up a lot of time. While insertion sort is a useful sort method, depending on the file type, it could take up a lot of time to sort big word files like 10e5. However, when compared to Bubble and Selection sort, it does outperform when it comes to sorting larger files like 10e5.

Explanation:
For selection sort, all of the word types had similar slopes. The way selection sort works is that it gets the biggest and lowest value, looks at their position, and sorts them if needed. It then does the same with second lowest value and does the same until it reachest the last lowest value. This is then repeated until the array is sorted. The advantage of Selection Sort is that it only sorts when necessary. The disadvantage with this is that for big word files like 10e5, it has to repeat that cycle multiple times, taking up too much time.
